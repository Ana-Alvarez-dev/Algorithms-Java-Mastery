# References

## Purpose

This document presents the primary academic references supporting the concepts discussed throughout the Searching module.

The selected bibliography consists of internationally recognised textbooks, official documentation and authoritative Computer Science resources.

Rather than relying on isolated internet articles, this module is based primarily on university-level literature that has shaped the study of algorithms, data structures and software engineering for several decades.

Students are encouraged to consult these references for deeper theoretical explanations, alternative implementations and broader algorithmic perspectives.

---

# Primary References

The following works constitute the principal academic foundation of this module.

---

## Introduction to Algorithms

**Thomas H. Cormen**

**Charles E. Leiserson**

**Ronald L. Rivest**

**Clifford Stein**

MIT Press

Often referred to simply as **CLRS**, this book is considered one of the most influential textbooks in Computer Science.

Relevant topics include:

- algorithm analysis;
- asymptotic complexity;
- searching algorithms;
- divide and conquer;
- correctness proofs;
- algorithm design techniques.

This repository adopts many of its methodological principles regarding algorithmic reasoning and complexity analysis.

---

## Algorithms

**Robert Sedgewick**

**Kevin Wayne**

Addison-Wesley Professional

This text provides both theoretical foundations and practical implementations of searching and sorting algorithms.

Relevant topics include:

- Sequential Search;
- Binary Search;
- Symbol Tables;
- Ordered Arrays;
- Algorithm Analysis;
- Practical Java implementations.

The engineering perspective presented throughout this repository is strongly influenced by this work.

---

## The Algorithm Design Manual

**Steven S. Skiena**

Springer

This book focuses on practical problem solving and algorithm selection.

Relevant topics include:

- recognising algorithmic patterns;
- selecting appropriate algorithms;
- problem-solving strategies;
- practical applications.

Its emphasis on reasoning before implementation aligns closely with the objectives of this repository.

---

## The Art of Computer Programming

**Donald E. Knuth**

Addison-Wesley

Although considerably more advanced,

Knuth's work remains one of the foundational references in algorithm analysis and computational reasoning.

Students interested in deeper mathematical analysis are encouraged to consult this series after mastering the material presented in this repository.

# Academic Integrity

The objective of this repository is educational.

Students are encouraged to:

- understand algorithms before implementing them;
- reason about correctness;
- analyse computational complexity;
- write original implementations;
- compare different algorithmic strategies;
- consult multiple academic sources.

Copying implementations without understanding their behaviour is strongly discouraged.

Algorithmic understanding should always precede programming.

---

# Engineering Perspective

Professional software engineers continuously consult technical references throughout their careers.

No engineer memorises every algorithm or API.

Instead, experienced professionals know:

- where to find reliable information;
- how to evaluate different approaches;
- how to compare alternative solutions;
- how to apply theoretical knowledge to practical problems.

Developing this habit is one of the primary objectives of this repository.
